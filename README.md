# Homework 2 | Deep Learning 2025-26

This section of the repository contains both the implementation and the generated results for Homework 2 of the Deep Learning course. 

The homework consists of two questions which involve programming. All code related to this homework, along with the plots, saved models, and result files produced during experimentation, is included in this directory.

Before running any of the scripts, it is recommended to create a dedicated virtual environment and install the required dependencies. The environment can be set up as follows:

```bash
python3 -m venv .venv
source .venv/bin/activate        # On macOS/Linux
# .venv\Scripts\activate.bat     # On Windows

pip install -r requirements.txt
```

The organization of the implementation follows the structure of the assignment and is detailed below.

## Question 1 - Image classification with CNNs
### 1. Logits vs Softmax
**File:** `hw2_q1.py`  
Implements a convolutional neural network to compare the use of **logits only** versus **logits followed by a Softmax layer** at the output.

### 2. Max Pooling Experiment
**File:** `hw2_q1.py`  
Implements an extension of the previous convolutional network by incorporating **MaxPooling** layers after each convolutional block.

---

Both experiments **(1.1 Logits vs Softmax and 1.2 Max Pooling)** are executed **within a single run** of the script.  
The code automatically evaluates the following four configurations:

- Logits without MaxPooling
- Softmax without MaxPooling
- Logits with MaxPooling
- Softmax with MaxPooling

## Question 2 - RNA Binding Protein (RBPs) Interaction Prediction
All programming components of Question 2 are implemented in:  
**Dir:** `hw2_q2/`  

### 1. Two different deep neural network architectures - CNN and RNN
Implements the CNN and RNN architecture models as well as the optimisation strategy.

This project implements two deep learning architectures: a **Convolutional Neural Network (CNN)** and a **Recurrent Neural Network (RNN)**, together with an automatic hyperparameter optimisation strategy.

---

#### Hyperparameter optimisation with Optuna

To determine the best hyperparameters for both models, we used **Optuna**.  
The Optuna optimisation script is implemented in:

**File:** `hw2_q2/hw2_q2_optuna.py`

> **NOTE:** This optimisation script can be executed locally using:
>
> ```bash
> cd hw2_q2
> python3 hw2_q2_optuna.py
> ```
>
> However, due to computational constraints, the Optuna hyperparameter search was executed on **Google Colab**.
> For this purpose, we used the following notebook:
>
> **File:** `hw2_q2/HW2_Q2.ipynb`

The hyperparameter search space is defined in:

**File:** `hw2_q2/config.py`

#### Optuna results

After running the Optuna optimisation, the best hyperparameters for each model are automatically saved in:

**Dir:** `hw2_q2/optuna_results/`

Specifically, Optuna generates the following files:
- `best_cnn_params.json`
- `best_rnn_params.json`

These files store the optimal hyperparameters found during the search.

---

#### Training the final models

Once the Optuna optimisation is completed, the final models can be trained using:

```bash
cd hw2_q2
python3 cnn_model.py
```

```bash
cd hw2_q2
python3 rnn_model.py
```

Each model automatically loads the corresponding best hyperparameters from the JSON files generated by Optuna (best_cnn_params.json and best_rnn_params.json) and trains the network using those optimal configurations.
